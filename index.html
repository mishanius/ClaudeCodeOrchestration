How I Built a Multi-Agent AI System That Runs My Dev Team

A Python orchestrator, Claude SDK, and an event-driven architecture that lets AI agents write code, manage projects, and talk to each other — autonomously.

The Big Picture

You send a Telegram message. A team of AI agents spins up — a Tech Lead breaks down the work, engineers write code in parallel, QA tests everything. You get progress updates. No babysitting.

Every message becomes a JSON event. The orchestrator picks it up, loads a warm Claude session, and the agent executes — reading files, running commands, writing code, or delegating to other agents.

Agents

Each agent is a Claude instance with persistent context, role-specific instructions, and isolated tools. They don't manage their own lifecycle — the orchestrator handles startup, sessions, failures, and communication.

Agents communicate through the same event system. The TL sends a message to an engineer — the orchestrator delivers it on the next cycle. All tasks live on Kanban boards, moving through TODO → In Progress → Done.

Tools (MCP)

Agents interact with the world through MCP servers — each one exposes a set of capabilities. Adding a new tool means writing a new server. The agent doesn't change. Most of these were built in-house:

external comms · web automation · AI image generation
  • Orchestration Tools — The orchestrator's core server. Inter-agent messaging (send_message), project/agent creation, Telegram & WhatsApp communication, voice generation, and image tools. Every agent gets this.
  • OpenProject MCP — Kanban boards, task creation, assignments, milestones, and status tracking. Agents manage their own tickets.
  • Memory MCP — Persistent memory across sessions. Hybrid keyword + semantic search over a SQLite + vector DB. Agents save decisions, lessons, and session summaries that survive restarts.
  • Telegram / WhatsApp — Direct communication with the human. Telegram bot for real-time messaging, WhatsApp via browser automation for file sharing.
  • Playwright — Browser automation for web interaction. Agents can navigate, screenshot, click, and scrape — used for research, monitoring, and external site interaction.
  • Image Generator — AI image generation via AWS Bedrock (Nova Canvas, Stable Diffusion). Sprites, diagrams, and creative assets on demand.

In Practice

I send: "Build a multiplayer battleships game with lobby system."

The Chief of Staff creates a project. The Tech Lead designs the architecture and creates 15 tickets across 6 milestones. Two engineers start coding in parallel. QA reviews the first PR. I get updates on Telegram throughout.

It's not perfect. Sessions corrupt. Agents misunderstand requirements. But it works — and it gets better every day.
